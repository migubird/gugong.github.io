# 故宫知识图谱 (gugong-kb)

> 用AI将故宫600年历史转化为可交互的空间-人物-事件知识图谱，让"走进一座宫殿"等于"打开一个时空入口"。

**在线阅读**: [https://migubird.github.io/gugong.github.io](https://migubird.github.io/gugong.github.io)

---

## 一句话介绍

把故宫从"静态建筑群"变成"活的历史数据库"——点击一座宫殿，看到谁在这里生活过、谁在这里执政、发生过什么事件，拖动时间滑块，看宫殿用途如何随朝代变迁。

---

## 核心成果（v0.2）

- **72 座宫殿**（外朝大殿、内廷后寝、东西六宫、附属宫殿、园林建筑、城门门禁等 9 大类别）
- **26 位皇帝**（明清两朝，完整时间线从永乐到宣统）
- **221 位人物**（大学士、尚书侍郎、将军、皇后、妃嫔、太监等）
- **82 个历史事件**（登基、政变、大婚、火灾、战争等）
- **727 条关系边**（居住/执政/君臣/父子/政敌/家族等）
- **交互式平面图**（上北下南，颜色编码分类，时间滑块，点击探索关联人物与事件）
- **皇帝时间线（地铁图）** + **知识图谱浏览器**
- **AI Agent Skills**（enrich / extract / map / validate 自动化知识管线）

## 快速体验

### 交互式故宫平面图

打开 [app/map/index.html](https://migubird.github.io/gugong.github.io/app/map/) 或克隆后本地打开：

```bash
# 本地预览
python3 -m http.server 8080
# 然后访问 http://localhost:8080/app/map/
```

点击任意宫殿，侧边栏会展示：
- 宫殿基本信息（建造年代、用途、尺寸）
- 关联皇帝（谁在此居住/执政）
- 关联人物（谁在此办公/生活）
- 关联事件（发生过什么）

## 数据结构

```
data/
├── ontology.json       # 本体定义（分类体系）
├── palaces.json        # 宫殿实体
├── emperors.json       # 皇帝实体
├── figures.json        # 人物实体（大臣/后妃等）
├── events.json         # 历史事件
├── relations.json      # 关系网络
└── timeline.json       # 时间线数据
```

所有数据均为 JSON 格式，方便 AI 管线处理。

## 与 shiji-kb 的关系

本方案仿造 [shiji-kb](https://github.com/baojie/shiji-kb) 的架构理念，但核心差异在于：

| 维度 | shiji-kb | gugong-kb |
|------|----------|-----------|
| 核心载体 | 文本（130章史记） | 空间（100+宫殿）+ 时间 |
| 可视化 | 语法高亮阅读器 + 地铁图 | 平面地图 + 地铁图 + 知识图谱 |
| 空间维度 | 无 | **核心维度**（平面坐标） |
